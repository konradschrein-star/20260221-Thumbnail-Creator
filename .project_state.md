# Project State: AI Thumbnail Rendering Farm

## Core Stack
- **Frontend**: Next.js 14, Tailwind CSS, TypeScript
- **Backend**: FastAPI, Python 3.11+, Pydantic
- **Queue**: Redis + RQ (Custom LIFO implementation)
- **Image Processing**: Pillow (PIL)
- **AI Services**: FAL AI (Background Generation), Qwen (Alternative)
- **Storage**: R2 (Cloudflare) or Local Storage
- **Database**: PostgreSQL

## Status
- **Phase 1: Audit & Scaffold**: Completed consolidation of initial Kimi AI codebase. Monorepo structure established. `CLAUDE.md` and `setup.ps1` created.
- **Next Phase**: Phase 2: Core API & LIFO Engine implementation verification and refinement.

## Active Configuration
- `LIFOQueue`: Custom class in `backend/app/queue/lifo_queue.py` providing strict LIFO/Stack semantics for priority jobs.
- `Compositor`: Deterministic Pillow-based rendering in `worker/app/compositor.py`.
